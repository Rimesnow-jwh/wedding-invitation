# 婚礼邀请函 H5 设计文档

> 蒋武华 & 彭斯 · 2026年8月8日

## 项目概述

制作一张「长图版」婚礼邀请函 H5 页面，功能对齐微信小程序「婚礼纪」核心体验：访客记录、祝福填写、赴宴回执、新人联系方式。同时支持一键导出 PNG 长图分享至朋友圈。

**设计风格：新中式雅致风** — 米白+暖杏+檀木+香槟金配色体系，传统东方美学克制升级，00后审美取向。

---

## 技术栈

| 层级 | 选型 | 说明 |
|------|------|------|
| 前端 | HTML + CSS + 原生 JS | 纯静态，零框架依赖，微信浏览器加载快 |
| 后端 | Supabase | PostgreSQL 数据库 + 自动 REST API，免费额度足够 |
| 部署 | Vercel | 免费部署，自带 HTTPS + 全球 CDN |
| 长图导出 | html2canvas | 前端渲染页面为 PNG 长图 |
| 字体 | 思源宋体（Noto Serif SC） | 免费可商用，Google Fonts CDN 加载 |

## 数据库设计（Supabase）

### visitors 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid (PK) | 自动生成 |
| name | text | 访客姓名 |
| guest_count | int | 赴宴人数 |
| phone | text | 联系电话 |
| attending | bool | 是否参加 |
| blessing | text | 祝福语（可空） |
| visitor_fingerprint | text | 浏览器指纹（去重统计浏览量） |
| created_at | timestamptz | 提交时间 |

**行级安全策略（RLS）**：允许匿名 INSERT 和 SELECT（读取祝福列表和统计数据）。

## 图片与背景融合方案

4种融合样式轮换使用，避免视觉疲劳：

| 样式 | 适用版块 | 实现方式 |
|------|----------|----------|
| **全幅满版** | 封面 | 图片占满宽度，底部 CSS 渐变遮罩过渡到米白背景 |
| **圆形晕影** | 邀请语、联系方式 | border-radius: 50% + box-shadow 柔焦，四周自然融入 |
| **侧边卡片** | 喜宴信息、赴宴回执 | 图片和文字左右错位排列，圆角 + 微弱阴影 |
| **网格相册** | 相册展示 | 2-3列网格，图片之间微间距，错落圆角 |

所有图片边缘使用 CSS `mask-image: linear-gradient(...)` 或叠加半透明渐变层实现与背景的自然过渡。

## 交互功能详情

### 1. 赴宴回执（RSVP）
- 表单字段：姓名、赴宴人数、联系电话、是否参加（单选）
- 提交后数据写入 `visitors` 表
- 页面实时显示「已有 X 人确认赴宴」
- 同一设备可修改之前的回执

### 2. 祝福留言墙
- 输入框 + 署名（默认使用回执中的姓名）
- 提交后写入 `visitors.blessing` 字段
- 留言列表按时间倒序，显示相对时间（如「3分钟前」）
- 显示「已收到 X 条祝福」

### 3. 访客记录
- 页面加载时生成浏览器指纹（Canvas fingerprint）
- 写入 `visitors` 表（attending 和 blessing 可为空）
- 显示「已有 X 人浏览过」
- 每日同一指纹不重复计数

### 4. 联系方式
- 展示新郎电话 13349676608（tel: 链接一键拨打）
- 展示新娘电话 18374893665
- 微信号展示（可复制）

## 长图导出

- 使用 `html2canvas` 库
- 页面底部固定「📸 保存长图」按钮
- 导出流程：
  1. 点击按钮
  2. 显示 loading 动画
  3. html2canvas 渲染整页 → 生成 PNG
  4. 导出时 CSS class 隐藏表单元素、按钮
  5. 弹出图片，用户长按保存到相册
